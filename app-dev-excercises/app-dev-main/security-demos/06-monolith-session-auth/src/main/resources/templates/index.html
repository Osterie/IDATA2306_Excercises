Welcome to our page. Here we demonstrate basic security handling for a monolith full-stack Spring Boot application. The following concepts are shown here:

  • Storing users in a database
  • Configuration of necessary authorization levels for different URLs (pages) of the application
  • Accessing currently logged-in user
  • Passing user data to Thymeleaf templates
  • Conditional elements in Thymeleaf - some elements in the navigation shown only for logged-in users, some only for admins
  • Several pages on the site: some requiring no authorization, some accessible to all users, some - only to admins
  • Including a reusable navigation component in several pages

You can try to access a profile page of another user. It will fail.
